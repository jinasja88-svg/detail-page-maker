# 멀티 상세페이지 조합 시스템 설계

> 경쟁사 상세페이지 N개를 크롤링하여, 각 섹션의 장점만 뽑아 조합하고,
> 저작권 이슈 없이 내 상품에 맞는 새로운 상세페이지를 만드는 시스템.

---

## 배경

### 기존 방식 (단일 상세페이지)
- URL 1개 크롤링 → 이미지 분할 → 섹션별 HTML 재현
- 문구: 참조 1에서 추출 / 디자인: 참조 2에서 추출
- 5개 배치 단위로 compare 파일 생성
- 문제점: 하나의 상페에만 의존 → 그 상페의 약한 부분도 그대로 따라감

### 업그레이드 방식 (멀티 조합)
- URL 3~5개를 동시에 크롤링
- AI가 모든 섹션을 분석하고 역할별로 분류
- 역할별 "베스트 섹션"을 선별하여 조합
- 카피는 여러 상페에서 좋은 것만 뽑아 변형 (저작권 회피)
- 디자인은 1번 상페(뼈대) 톤앤매너로 통일
- AI가 CRO 관점에서 최적 순서로 배열
- 실제 생성과 검토는 반드시 `5개 배치 단위`로 유지

---

## 전체 파이프라인

```
입력: URL 3~5개 + 내 상품 정보
  ↓
Phase 1. 멀티 크롤링 + 분할
  ↓
Phase 2. 전체 섹션 역할 태깅
  ↓
Phase 3. 역할별 베스트 섹션 선택
  ↓
Phase 4. 최적 순서 배열 → ★ 사용자 확인 ★
  ↓
Phase 5. 카피 변형 (저작권 회피 + 내 상품 반영)
  ↓
Phase 6. 디자인 통일 (1번 상페 톤앤매너, 5개 배치 생성)
  ↓
Phase 7. 내 제품 사진 교체 + 최종 HTML 출력
```

### 5개 배치 원칙
- 멀티 조합이어도 한 번에 전체 섹션을 생성하지 않음
- 배열 확정 후 실제 compare 생성은 `1~5`, `6~10`처럼 5개씩 진행
- 각 배치가 끝날 때마다 누적 `combined_review.html`을 갱신
- 사용자는 배치마다 결과를 검토한 후 다음 배치로 진행
- 품질이 흔들리면 다음 배치로 넘어가지 않음

## Phase 1. 멀티 크롤링 + 분할

### 입력
```
urls:
  - url: "https://smartstore.naver.com/xxx/products/111"
    label: "A"
    role: "뼈대"          # 이 상페의 디자인 톤앤매너를 기준으로 삼음
  - url: "https://smartstore.naver.com/yyy/products/222"
    label: "B"
    role: "참조"
  - url: "https://www.coupang.com/vp/products/333"
    label: "C"
    role: "참조"
```

### 처리
- 기존 `crawler.py`로 각 URL을 순차 크롤링
- 기존 `image_splitter.py`로 각각 이미지 분할
- 결과 폴더 구조:
```
output/
  multi_job_20260330/
    A_naver_xxx/
      detail_images/
        detail_001.jpg
        detail_002.jpg
        ...
    B_naver_yyy/
      detail_images/
        detail_001.jpg
        ...
    C_coupang_333/
      detail_images/
        detail_001.jpg
        ...
    config.json          # 입력 URL, label, role 정보
```

      "label": "B",

      "role": "참조",

## Phase 2. 전체 섹션 역할 태깅

### 목적
모든 크롤링된 섹션 이미지를 AI에게 보여주고, 각 섹션이 어떤 역할인지 분류한다.

### 역할 태그 목록
| 태그 | 설명 | 예시 |
|------|------|------|
| `HOOK` | 첫인상, 메인 배너, 핵심 한줄 카피 | "피부가 달라지는 28일" |
| `PAIN` | 고객 고민/문제 제기 | "이런 고민 있으신가요?" |
| `SOLUTION` | 해결책 제시, 제품 소개 | "OO가 해결해드립니다" |
| `FEATURE` | 제품 특징, 성분, 스펙 | 성분표, 기능 나열 |
| `BENEFIT` | 사용 시 기대효과 | "사용 후 피부 변화" |
| `SOCIAL_PROOF` | 후기, 수상, 인증, 언론 | 리뷰 캡처, 인증마크 |
| `HOW_TO` | 사용법, 활용법 | "하루 2회, 세안 후 사용" |
| `COMPARE` | 경쟁제품 비교, 전후 비교 | Before/After, 비교표 |
| `INGREDIENT` | 원료, 성분 상세 설명 | "히알루론산이란?" |
| `BRAND` | 브랜드 스토리, 철학 | "10년간 연구해온..." |
| `GUARANTEE` | 배송, AS, 환불, 보증 | "100% 환불 보장" |
| `BUNDLE` | 구성품, 패키지 설명 | "세트 구성 안내" |
| `CTA` | 구매 유도, 할인, 한정 | "지금 주문하면 30% 할인" |
| `FILLER` | 장식, 구분선, 의미 없는 여백 | 스킵 대상 |

### AI 프롬프트 (섹션 분석용)
```
아래 이미지는 상세페이지의 한 섹션입니다.

1. 역할(role): 위 태그 목록 중 가장 적합한 것 1개
2. 카피 텍스트: 이미지 안에 보이는 텍스트를 모두 추출
3. 카피 품질 점수(copy_score): 1~10
   - 설득력, 구체성, 감정 자극, 차별화 기준
4. 디자인 품질 점수(design_score): 1~10
   - 레이아웃, 가독성, 시각적 임팩트, 완성도 기준
5. 디자인 요소: 배경색, 텍스트 색/크기/위치, 레이아웃 패턴
6. 핵심 메시지 요약: 이 섹션이 전달하려는 핵심 1줄
```

### 출력: section_analysis.json
```json
{
  "sections": [
    {
      "id": "A_013",
      "source": "A",
      "file": "A_naver_xxx/detail_images/detail_013.jpg",
      "role": "HOOK",
      "copy_text": "자연에서 찾은 피부 해답",
      "copy_score": 8,
      "design_score": 9,
      "design_elements": {
        "bg_color": "#1a1a1a",
        "text_color": "#ffffff",
        "layout": "center_text_over_image",
        "font_size": "large"
      },
      "core_message": "자연 성분 기반 스킨케어 브랜드 소개"
    },
    {
      "id": "B_003",
      "source": "B",
      "file": "B_naver_yyy/detail_images/detail_003.jpg",
      "role": "HOOK",
      "copy_text": "28일의 기적, 당신의 피부가 증명합니다",
      "copy_score": 9,
      "design_score": 7,
      "design_elements": {
        "bg_color": "#f5f0eb",
        "text_color": "#333333",
        "layout": "left_text_right_image",
        "font_size": "large"
      },
      "core_message": "28일 사용 후 효과를 강조하는 후킹"
    }
  ]
}
```

### 비용 참고
- 기존 섹션 분석: GPT-4o 기준 약 9원/섹션
- 3개 상페 x 평균 20섹션 = 60섹션 = 약 540원

## Phase 3. 역할별 베스트 섹션 선택

### 방식
1. `section_analysis.json`에서 같은 역할(role)끼리 그룹핑
2. 각 그룹에서 **카피 베스트** / **디자인 베스트** 선정
3. 조합 규칙:
   - 카피 베스트 = 디자인 베스트 → 그 섹션 그대로 사용
   - 카피 베스트 ≠ 디자인 베스트 → **카피는 카피 베스트에서, 디자인은 디자인 베스트에서** 크로스 조합
   - `FILLER` 역할은 자동 제외
   - 뼈대(A) 상페에만 있는 역할은 A 것 그대로 사용

### 출력: best_selection.json
```json
{
  "selections": [
    {
      "role": "HOOK",
      "copy_from": "B_003",
      "copy_text": "28일의 기적, 당신의 피부가 증명합니다",
      "design_from": "A_013",
      "design_ref_image": "A_naver_xxx/detail_images/detail_013.jpg",
      "reason": "B의 카피가 더 구체적(28일), A의 디자인이 더 임팩트 있음"
    },
    {
      "role": "PAIN",
      "copy_from": "C_005",
      "copy_text": "건조함, 당김, 칙칙함... 매일 반복되는 피부 고민",
      "design_from": "A_015",
      "design_ref_image": "A_naver_xxx/detail_images/detail_015.jpg",
      "reason": "C의 카피가 고민을 가장 구체적으로 나열, A 디자인 통일"
    }
  ]
}
```

### 사용자 확인 포인트
- 이 단계에서 선택 결과를 **리뷰 HTML**로 보여줌
- 각 역할별로 "카피 출처 / 디자인 출처 / 이유" 표시
- 사용자가 수동으로 교체 가능 (예: "PAIN은 B_007로 바꿔줘")
- 전체 후보를 보여주되, 이후 실제 생성은 5개 배치 단위로만 진행

## Phase 4. 최적 순서 배열

### 기본 순서 템플릿 (CRO 기반)
```
1. HOOK        — 첫인상, 핵심 카피
2. PAIN        — 고객 고민 자극
3. SOLUTION    — 해결책 = 내 제품
4. FEATURE     — 제품 핵심 특징 (3개 이내)
5. BENEFIT     — 사용 시 기대효과
6. SOCIAL_PROOF — 후기/인증으로 신뢰
7. INGREDIENT  — 성분/원료 상세 (해당 시)
8. HOW_TO      — 사용법
9. COMPARE     — 비교 우위 (해당 시)
10. BRAND      — 브랜드 신뢰 (해당 시)
11. BUNDLE     — 구성/패키지
12. GUARANTEE  — 배송/환불/보증
13. CTA        — 구매 유도
```

### 배열 규칙
- 위 순서를 기본 뼈대로 사용
- 해당 역할의 베스트 섹션이 없으면 스킵
- 같은 역할 섹션이 2개 이상 선택된 경우 → 점수 높은 순으로 연속 배치
- 뼈대(A) 상페의 원래 순서를 최대한 존중하되, CRO 순서와 충돌하면 CRO 우선
- AI가 순서를 정한 후 **배열 이유를 함께 출력**

### 출력: arrangement.json
```json
{
  "arrangement": [
    {
      "order": 1,
      "role": "HOOK",
      "section_id": "B_003",
      "copy_from": "B_003",
      "design_from": "A_013",
      "reason": "강한 숫자 카피(28일)로 시작하여 즉시 관심 유도"
    },
    {
      "order": 2,
      "role": "PAIN",
      "section_id": "C_005",
      "copy_from": "C_005",
      "design_from": "A_015",
      "reason": "고민 나열 후 해결책으로 자연스러운 전환 유도"
    }
  ]
}
```

### 사용자 확인 포인트
- 배열안을 **썸네일 리스트 HTML**로 보여줌
- 각 칸에: 순서번호 + 역할 태그 + 카피 출처 + 디자인 출처 + 썸네일
- 사용자가 드래그 또는 지시로 순서 변경 가능
- **이 확인을 거친 후에만 Phase 5로 진행**
- 배열 확정 후에는 `batch_plan` 기준으로 5개씩 나눠 처리

## Phase 5. 카피 변형

### 목적
- 원본 카피를 그대로 사용하면 저작권 문제 발생
- 핵심 메시지는 유지하되 표현을 변형
- 내 상품 정보를 반영

### 변형 규칙
1. **핵심 메시지 유지**: "28일 후 피부 변화" → 메시지 동일 OK
2. **표현 변형 필수**: 문장 구조, 어순, 비유, 수식어를 바꿈
3. **내 상품명/성분/특징 대입**: 경쟁사 제품명 → 내 제품명
4. **숫자/데이터 변경**: 내 상품에 맞는 실제 수치 사용
5. **원본 문장 3어절 이상 연속 동일 금지**: 저작권 안전장치

### AI 프롬프트 (카피 변형용)
```
아래는 경쟁사 상세페이지에서 추출한 카피입니다.
이 카피의 핵심 메시지와 설득 구조는 유지하되,
표현을 완전히 새롭게 작성해주세요.

[원본 카피]
"28일의 기적, 당신의 피부가 증명합니다"

[내 상품 정보]
- 상품명: OOO 에센스
- 핵심 성분: 히알루론산, 나이아신아마이드
- 핵심 강점: 72시간 보습 지속
- 타겟: 30대 건성 피부

[규칙]
- 원본 문장을 그대로 쓰지 말 것
- 3어절 이상 연속 동일 금지
- 내 상품의 실제 특징을 반영할 것
- 같은 설득 구조(숫자+효과+증명)를 활용하되 새 문장으로
```

### 출력: copy_variants.json
```json
{
  "variants": [
    {
      "order": 1,
      "role": "HOOK",
      "original_copy": "28일의 기적, 당신의 피부가 증명합니다",
      "original_source": "B_003",
      "new_copy": "72시간 촉촉함, 당신의 손끝이 먼저 알아챕니다",
      "persuasion_structure": "숫자 + 효과 + 감각적 증명"
    },
    {
      "order": 2,
      "role": "PAIN",
      "original_copy": "건조함, 당김, 칙칙함... 매일 반복되는 피부 고민",
      "original_source": "C_005",
      "new_copy": "세안 직후부터 시작되는 당김, 오후만 되면 올라오는 각질... 매일이 같은 고민",
      "persuasion_structure": "구체적 상황 나열 + 공감"
    }
  ]
}
```

## Phase 6. 디자인 통일

### 목적
- 여러 상페에서 가져온 섹션들이 각각 다른 디자인이면 어색함
- 1번 상페(뼈대)의 톤앤매너를 기준으로 전체를 통일

### 뼈대에서 추출할 디자인 시스템
```json
{
  "design_system": {
    "primary_bg": "#ffffff",
    "secondary_bg": "#f5f0eb",
    "accent_bg": "#1a1a1a",
    "primary_text": "#333333",
    "accent_text": "#c8956c",
    "heading_font": "bold, 28-36px",
    "body_font": "regular, 16-18px",
    "caption_font": "light, 13-14px",
    "layout_patterns": [
      "center_text_over_image",
      "left_text_right_image",
      "icon_grid_3col",
      "full_width_banner"
    ],
    "spacing": "섹션 간 40-60px",
    "corner_radius": "0px (직선 스타일)"
  }
}
```

### 처리 방식
- 기존 시스템의 섹션 분석(Claude 비전) + HTML 재현을 그대로 사용
- 단, 디자인 참조 이미지는 뼈대(A) 상페의 해당 섹션 사용
- 카피는 Phase 5에서 변형된 것을 삽입
- 기존과 동일하게 `contenteditable`로 텍스트 수정 가능
- compare 파일 형식: `왼쪽 = 디자인 참조 원본 / 오른쪽 = 새로 생성된 HTML`
- 생성 순서는 `arrangement.json`을 따르되, 실제 작업은 5개 배치 단위로 끊어서 진행
- 각 배치가 끝날 때마다 누적 `combined_review.html`을 다시 생성

### 출력 구조
```
output/
  multi_job_20260330/
    section_01_HOOK_compare.html
    section_02_PAIN_compare.html
    section_03_SOLUTION_compare.html
    ...
    combined_review.html
    design_system.json
```

## Phase 7. 제품 사진 교체 + 최종 출력

### 제품 사진 입력
- 사용자가 `product_photos/` 폴더에 제품 사진을 넣음
- 여러 각도: 정면, 측면, 사용 중, 디테일 등

### 교체 방식
- 기존과 동일: DALL-E 또는 Gemini로 제품 이미지 생성/교체
- HTML 내 제품 이미지 영역에 내 제품 사진 삽입

### 최종 출력물
1. **개별 섹션 compare HTML**: 왼쪽 디자인 참조 / 오른쪽 최종 결과
2. **combined_review.html**: 전체 상세페이지 미리보기
3. **final_page.html**: 최종 상세페이지 (compare 없이 결과물만)
4. **텍스트 분리 JSON**: 모든 텍스트를 이미지와 별도로 편집 가능
