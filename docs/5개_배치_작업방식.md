# 5개 배치 작업 방식

## 목표
- 상세페이지 장수가 많아도 `5개 단위`로 끊어서 작업한다.
- 각 배치가 끝날 때마다 compare HTML과 combined review를 갱신한다.
- 사용자는 항상 누적 결과를 한 파일에서 바로 검토할 수 있어야 한다.

## 기본 흐름
1. URL을 크롤링해서 `detail_001`, `detail_002` 같은 상세 이미지를 추출한다.
2. 전체 상세 이미지를 `1~5`, `6~10`, `11~15`처럼 5개 단위 배치로 나눈다.
3. 현재 배치의 각 이미지를 `왼쪽 원본 / 오른쪽 HTML 재현` compare 파일로 만든다.
4. 배치 5개가 끝나면 cumulative combined review 파일을 다시 만든다.
5. 다음 배치로 넘어가 동일한 과정을 반복한다.

## 출력 원칙
- 개별 파일: `slug_detail_005_compare.html`
- 누적 파일: `slug_combined_review.html`
- 배치 계획서: `slug_batch_plan.md`
- 배치 데이터: `slug_batch_plan.json`

## 현재 자동화 범위
- 상세 이미지 개수 확인
- 5개 단위 배치 계획 생성
- 이미 만들어진 compare 파일 기준으로 누적 combined review 생성
- 다음 작업 배치 계산

## 실행 예시
```powershell
python src\compare_batch_manager.py `
  --crawl-dir output\curigen_cafe24_com_b624375f `
  --compare-dir output\test_single_section `
  --slug curigen `
  --product-label "Curigen" `
  --batch-size 5 `
  --resume-after-last-done `
  --combined-file curigen_combined_review.html `
  --plan-md curigen_batch_plan.md `
  --plan-json curigen_batch_plan.json
```

## 산출물 예시
- `output/test_single_section/curigen_combined_review.html`
- `output/test_single_section/curigen_batch_plan.md`
- `output/test_single_section/curigen_batch_plan.json`

## 수작업이 필요한 부분
- 각 섹션 오른쪽 HTML 재현의 실제 디자인
- 문구 조정
- 섹션별 높이 보정

## 큐리젠 현재 적용 방식
- compare 파일 prefix: `curigen`
- crawl folder: `output/curigen_cafe24_com_b624375f`
- compare folder: `output/test_single_section`
- 005~025까지 완료 후 combined를 누적 갱신하는 방식으로 진행 중
