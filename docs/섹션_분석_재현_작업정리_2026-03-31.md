# 섹션_분석_재현_작업정리_2026-03-31

이 문서는 현재 멀티 상세페이지 작업에서 `섹션 분석 -> 텍스트 자산화 -> 샘플 재현`까지 진행한 내용을 정리한 최신 기준 문서입니다.

## 현재 목표

기존처럼 상세페이지를 `대충 비슷하게` 만드는 것이 아니라,
각 섹션을 아래 3가지 축으로 분리해서 관리합니다.

1. `layout spec`
- 재현용 좌표/배치 데이터

2. `text assets`
- 원문 텍스트와 변형 규칙 데이터

3. `seedream analysis`
- 나중에 이미지 생성용으로 쓸 장면 분석 데이터

## 비용 기준

현재 추정 기준:
- 분석 모델 기본: `GPT-5.4 mini`
- 어려운 섹션만 `GPT-5.4`로 fallback

대략 추정:
- `레이아웃 분석 + Seedream 분석` 기준
- `GPT-5.4 mini`: 페이지당 약 20원 수준
- `GPT-5.4`: 페이지당 약 60원대 수준

26페이지 전체 대략:
- `mini`: 약 530원
- `GPT-5.4`: 약 1,650원

## 앞으로의 작업 원칙

1. 섹션은 2개씩 또는 많아도 3개씩 분석/생성
2. 텍스트 없는 영역엔 문구 억지 생성 금지
3. `layout spec`, `text assets`, `seedream analysis`를 분리 저장
4. 나중에 API 연결 시
   - 1차 `GPT-5.4 mini`
   - 실패/복잡 섹션만 `GPT-5.4`
